# Code of Conduct

## Purpose

The SAP Content Risk Playbook is an educational resource intended for professional use by enterprise IT leaders, SAP architects, and service providers. This Code of Conduct establishes expectations for how this repository should be used and discussed.

---

## Expected Use

### ✅ Encouraged
- **Professional Discussion**: Thoughtful questions and insights on SAP content risk
- **Educational Sharing**: Referencing this playbook in training, presentations, or documentation
- **Problem-Solving**: Using this content to inform migration decisions and risk assessments
- **Attribution**: Proper citation when referencing concepts or frameworks from this playbook

### ❌ Prohibited
- **Misrepresentation**: Claiming authorship or repackaging without attribution
- **Operational Misuse**: Attempting to execute remediation without qualified expertise
- **Harassment**: Demanding free consulting or aggressive solicitation
- **Bad Faith Forking**: Creating derivative works that misrepresent PythonMate's intent or expertise

---

## Interaction Standards

### When Requesting Clarification
- Provide specific context (SAP release, landscape size, industry)
- Reference the specific section or concept you are asking about
- Respect that this is documentation, not live support

### When Providing Feedback
- Be constructive and specific
- Assume good intent
- Understand that not all suggestions will be incorporated

### When Sharing Publicly
- Link to the original repository when possible
- Preserve context (don't quote out of context)
- Respect that PythonMate maintains editorial control

---

## What This Is Not

This repository is **not**:
- A community forum (no discussion threads)
- A support channel (no guaranteed response times)
- A collaborative wiki (no open editing)

It is a **curated knowledge artifact** maintained by a single source for quality and consistency.
